Component model

What each subsystem owns

Control plane

The CLI is the control plane. It decides which workflow to run, loads `.env`, refuses spend-capable commands unless `--live` is present, and wires all service clients together. The dashboard is deliberately excluded from this path so viewing evidence cannot mutate chain state.

Data plane

The data plane is the Jito and Yellowstone path. Jito accepts the signed bundle and reports bundle-level status. Yellowstone observes the transaction signature and slot commitment progression so the final lifecycle log is stream-confirmed rather than RPC-only.

Failure strategy

How the stack reasons about bad outcomes

Failure Where it comes from What the stack does
expired_blockhash Block height exceeds the signed transaction's last valid block height. Builds evidence, asks the AI agent, refreshes blockhash only if the agent returns a valid retry decision.
fee_too_low Jito auction or bundle status indicates the tip was not competitive. Classifies the reason and preserves the live tip context for the next decision.
compute_exceeded Simulation or runtime logs show compute budget exhaustion. Records the fault so the operator can distinguish program/resource issues from network issues.
bundle_failure Jito reports failed, invalid, or timed-out bundle status. Stops treating the attempt as a success and writes the failure to the lifecycle log.
stream_timeout Yellowstone does not observe the signature lifecycle in time. Fails closed. The stack does not invent a processed/confirmed/finalized state.

AI agent contract

The model owns the retry decision, not the transaction signing surface

Agent input

  • failure message and classification context
  • signature and bundle id when available
  • fault mode
  • current slot and Jito leader window
  • live tip quote and tip-floor sample
  • blockhash, last valid block height, and current block height
  • Jito status and simulation logs when available

Agent output

  • `failure_classification`
  • `retry_action`
  • `blockhash_strategy`
  • `tip_lamports`
  • `wait_slots`
  • `confidence`
  • `reasoning_summary`
